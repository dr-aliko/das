Integration Recipe

1. Tailwind config (one-time)

Paste the tailwind.config = {…} block at the top of this file into your existing config under theme.extend. Adds ink-*, steel-*, semantic tints, max-w-page, and the card shadows. Removes nothing.

2. Replace the purple banner

In your base template (likely templates/base.html) wrap the existing purple header with class="md:hidden" (keep it for mobile) and add the new <header class="hidden md:block ..."> from §02 above it. Two distinct headers, one media query — done.

3. Page wrapper

Every page body gets <main class="mx-auto max-w-page md:px-6">. Inside it, page-specific containers can override with max-w-prose-wide for read-heavy content (exam lists, profile).

4. The 8/4 split (most common layout)

<div class="grid grid-cols-1 lg:grid-cols-12 gap-6">
  <div class="lg:col-span-8 space-y-4">
    <!-- main content stack -->
  </div>
  <aside class="lg:col-span-4 space-y-4 lg:sticky lg:top-20 lg:self-start">
    <!-- sidebar -->
  </aside>
</div>

Used in: Student Dashboard, Coach Student Snapshot. Note lg:sticky lg:top-20 — the sidebar stays in view as the main column scrolls. Mobile: stacks vertically.

5. Roster / card grid

<div class="grid grid-cols-1 md:grid-cols-2 xl:grid-cols-3 gap-3">
  <!-- DAS_card components -->
</div>

Coach roster, exam list, branş cards. Densifies as viewport grows.

6. Mobile non-regression — guaranteed by construction

Every class that changes desktop layout is prefixed: md: or lg:. The default classes (e.g. grid-cols-1, space-y-4) preserve mobile stacking. The only mobile-side change is the color palette — and that's intentional and applies app-wide via tokens.css already.

7. Suggested rollout

  1. Ship the Tailwind config additions + new app bar behind desktop_v2 flag.
  2. Migrate one page at a time, in this order: Profile → Coach Invitations → Coach Roster → Student Dashboard → Coach Snapshot → Branş Hub → Denemeler list → Tasks.
  3. Profile/Invitations are smallest — use them to validate the system on real Django templates first.
  4. Tasks/Snapshot are biggest — save for sprint 2.
